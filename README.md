專案名稱：
每日製程良率自動彙整與報告產出系統

改善問題：
目前每日製程良率資料需由人員登入報表系統下載，再以 Excel 手動篩選、分類、計算及製作圖表，最後複製到日報或管理報告中。

人工處理存在以下問題：

1. 每日需重複執行資料整理與製圖作業，耗費人力與時間。
2. 不同人員可能採用不同統計方式，容易產生計算差異。
3. 日、週、月良率需分別整理，資料更新與維護不易。
4. 手動複製公式、篩選資料及更新圖表時，可能發生漏列、誤植或引用範圍錯誤。
5. 報告無法快速依製程站點或 OPNO 比較良率趨勢。
6. 公司電腦受軟體安裝限制，無法直接部署 Python、Node.js 或其他外部程式。

目前作法：

1. 由人員從報表系統手動下載 ExportExcel 格式的良率資料。
2. 開啟下載的 Excel／XLS 檔案。
3. 依製程站點、日期或 OPNO 進行資料篩選。
4. 使用 Excel 公式或人工計算投入數、NG 數及良率。
5. 分別整理日、週、月良率。
6. 手動建立趨勢圖或比較圖。
7. 將統計結果及圖表貼入每日或定期報告。
8. 每次下載新資料後，需重新執行上述作業。

創新方案：
建立一套以 Excel VBA 巨集為核心的「每日良率自動彙整與報告產出工具」，不需安裝額外軟體，即可在公司既有 Microsoft Excel 環境中執行。

主要功能如下：

1. 自動尋找指定資料夾中最新的 ExportExcel*.xls 系統匯出檔。
2. 自動將系統資料匯入 RawData 工作表。
3. 以 OPNO 作為製程站點的主要分類依據，不使用較廣泛的 DATATYPE 分類。
4. 自動建立日、週、月三種統計層級。
5. 將日、週、月資料整合在同一張 PivotData 工作表。
6. 自動彙總以下資料：
    ・Total 片數
    ・NG 片數
    ・Total 顆粒數
    ・NG 顆粒數
    ・片良率
    ・顆粒良率
7. 良率採總數加權重新計算，避免直接平均各筆良率造成誤差。
8. 自動建立或更新樞紐分析表及樞紐圖。
9. 使用者只需下載資料並執行巨集，即可完成每日更新。
10. 程式採 ASCII／英文 VBA 程式碼，中文欄位使用 ChrW 函數組合，避免 VBA 匯入時發生中文亂碼。
11. 增加安全關檔、執行步驟提示及錯誤碼顯示功能，提高公司不同 Office 環境中的執行穩定性。
12. 後續可延伸為自動將圖表更新至固定日報或 PowerPoint 報告模板。

良率計算方式：

片良率：
（Total 片數合計－NG 片數合計）÷ Total 片數合計

顆粒良率：
（Total 顆粒數合計－NG 顆粒數合計）÷ Total 顆粒數合計

相關數據：

1. 測試資料筆數：349 筆。
2. 測試資料期間：2026/04/20～2026/05/19。
3. 原始資料站點類型包含：
    3DL、AOI、AVI、AXI、DB、FNT、SPI。
4. 系統原始資料主要欄位包含：
    RECORDDATE、DATATYPE、PRODUCTNAME、PRODUCTNO、EQP_ID、OPNO、MONOTYPE、Total Yield、Total 片數、NG 片數、Total 顆粒及 NG 顆粒。
5. 統計分類已由 DATATYPE 修正為 OPNO，以符合實際製程站點管理需求。
6. 已完成日、週、月良率整合資料結構。
7. 已完成自動抓取最新系統匯出檔的功能設計。
8. 已完成 Excel VBA 巨集版本，以因應公司無法安裝外部程式的限制。
9. 已針對 VBA 中文亂碼、來源活頁簿重複關閉、Automation Error 及錯誤訊息無法顯示等問題進行改善。

預期效益：

1. 縮短每日良率報表整理時間。
2. 降低人工篩選、公式複製及資料彙總錯誤。
3. 統一日、週、月良率的計算邏輯與報表格式。
4. 使用加權良率計算，提升跨日期與跨批次統計的正確性。
5. 可快速依 OPNO 比較不同製程站點的良率趨勢。
6. 透過樞紐分析表及樞紐圖，提高資料分析彈性。
7. 使用者只需下載資料及執行巨集，降低操作門檻。
8. 不需安裝 Python、Node.js 或額外套件，符合公司資訊安全及軟體管制要求。
9. 固定資料結構有利於後續串接日報、週報、月報及管理簡報。
10. 可將人員由重複性的報表整理工作，轉移至異常分析與改善活動。

建議後續量化指標：
・原人工整理時間：待實測填寫，例如每日 30～60 分鐘。
・自動化後處理時間：預估縮短至 1～5 分鐘。
・每月節省工時：依每日執行次數及實際工時換算。
・報表資料錯誤率：以導入前後錯誤或重工次數比較。
・報告產出準時率：比較導入前後每日報告完成時間。

專案摘要說明：
本專案針對每日製程良率資料需人工下載、整理、統計、製圖及貼入報告的重複性作業，開發一套以 Excel VBA 為核心的自動彙整工具。系統可自動讀取指定資料夾中最新的良率匯出檔，以 OPNO 區分製程站點，並依日、週、月三種層級彙總投入數、NG 數、片良率及顆粒良率。所有統計結果整合於同一份樞紐資料中，可直接更新樞紐分析表與趨勢圖。此方案不需安裝 Python或其他外部軟體，符合公司資訊安全及軟體管制環境，預期可大幅縮短報表製作時間、降低人工錯誤，並建立一致且可持續更新的製程良率管理機制。
